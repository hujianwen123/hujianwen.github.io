git submodule 情况处理

当你的项目里面包含两个或两个以上的仓库地址时，提交代码会失败,比如我这个hexo项目，引入了别人的仓库代码，报错如下：

这个时候你只要删除掉themes/butterfly子模块
删除之后再添加进去就行了，后续提交成功

本地切换远程仓库地址

有时候我们的本地项目需要切换远程项目地址
先删除本地的所有远程仓库连接，再添加你想要连接的新地址

修改远程github的默认分支

打开github的setting设置模块，里面可以修改默认分支
默认分支主要是影响到git clone 所拉取的内容
